# Workout & Diet Tracking App - Toss 스타일 재설계 기술 문서

## 1. 프로젝트 개요 (Project Overview)

이 프로젝트는 **Toss 스타일 디자인 철학**을 따르는 운동 및 식단 추적 앱의 전면 재설계입니다. 사용자 경험을 최우선으로 고려하여 기존 버그를 수정하고, 모바일 최적화, 새로운 기능을 추가하여 실무적인 웹 앱을 구현했습니다.

### 1.1 개발 목표 (Development Goals)
- 사용자 경험(UX) 최선화 및 고품질 애니메이션 적용
- 모바일 및 데스크탑 스크롤 기반 UI 최적화
- 상태 지속성 보장 및 데이터 정밀도 향상
- 풀스택 기능 확장 (사용자 프로필, 히스토리 관리)

### 1.2 기술 스택 (Technology Stack)
- **프론트엔드 (Frontend)**: React 18, TypeScript, Zustand, Material UI, Framer Motion
- **백엔드 (Backend)**: Java 21, Spring Boot 3.4, Spring Data JPA, Spring Security (JWT), MySQL

## 2. 구현된 기능 (Implemented Features)

### 2.1 사용자 피드백 및 대화 상자 (Improved Feedback)
- **Toss 스타일 다이얼로그**: 기존의 투박한 `window.confirm`을 대체하는 `ConfirmDialog` 컴포넌트 구현. 부드러운 라운드 처리와 Toss 고유의 버튼 스타일 적용.
- **통합 스낵바**: 삭제, 복사, 저장 등 모든 주요 액션에 대해 MUI Snackbar를 통한 직관적인 피드백 제공 ("삭제 완료", "클립보드에 복사되었습니다" 등).

### 2.2 고정밀 및 하이브리드 입력 시스템 (Workout Log)
- **운동 카테고리화**: 운동 추가 시 부위별(가슴, 등, 하체 등) 카테고리 선택 기능을 추가하여 수많은 종목 중 원하는 운동을 빠르게 검색 가능.
- **하이브리드 셀렉터**: 
  - **휠 피커**: 가로/세로 휠을 통한 직관적인 범위 선택.
  - **키보드 입력**: 직접 숫자를 입력하여 정확한 수치 기입 가능.
  - **증감 버튼**: 무게(+/- 5kg), 횟수(+/- 1회) 버튼을 통한 미세 조정 지원.
- **정밀도**: 무게는 최대 1000kg까지 0.5kg 단위로 조절 가능.

### 2.3 상세 데이터 요약 및 오류 진단 (Enhanced Summary)
- **풍부한 요약 정보**: 사용자 신체 정보(나이, 골격근량 등), 세트별 운동 기록(100kg x 10회), 식사별 음식 리스트 및 영양소 정보를 모두 포함.
- **상세 오류 메시지**: 요약 복사 실패 시 단순히 "실패"가 아닌, 어떤 정보(프로필, 운동, 식단 등)를 불러오지 못했는지 구체적인 원인을 스낵바로 표시.

### 2.4 풀스택 데이터 관리 (Full Stack)
- **기록 삭제 기능**: 백엔드 DELETE API와 프론트엔드 히스토리 페이지를 연동하여 잘못된 기록을 즉시 삭제 가능.
- **사용자 프로필**: 백엔드 `UserProfile` 엔티티와 연동된 `Settings` 페이지를 통해 지속적인 신체 변화 관리.

## 3. 주요 컴포넌트 및 훅

- **`ConfirmDialog`**: Toss 디자인 시스템을 반영한 범용 확인창.
- **`useScrollDirection`**: 스크롤 방향에 따라 상/하단 UI를 숨기거나 보여주는 최적화된 훅.
- **`Vertical/HorizontalScrollSelector`**: 고성능 스냅 스크롤이 적용된 휠 피커.
